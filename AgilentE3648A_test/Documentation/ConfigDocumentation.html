GpibPowerSupply

 Configuration attributes:
  • board (OpcUa_Int16) mandatory
    DOC Specifies which GPIB interface board the device is connected to.

  • primaryAddress (OpcUa_Int16) mandatory
    DOC Specifies the GPIB primary address of the device to be opened.

  • secondaryAddress (OpcUa_Int16) optional (the default value shown below will be used if non-present)
    DOC Specifies the GPIB secondary address of the device to be opened. If not specified the default value NO_SAD will be used.
    Default value:-1

  • timeOut (OpcUa_Int16) optional (the default value shown below will be used if non-present)
    DOC Specifies the timeout for i/o operations. If not specified the default value T10s will be used.
    Default value:-1

  • send_eoi (OpcUa_Int16) optional (the default value shown below will be used if non-present)
    DOC If nonzero the EOI line will be asserted with the last byte sent during writes. If not specified the default value 0 will be used.
    Default value:0

  • eos (OpcUa_Int16) optional (the default value shown below will be used if non-present)
    DOC Specifies the end-of-string character and whether or not its reception should terminatereades. If not specified the default value 1 will be used. More info at https://linux-gpib.sourceforge.io/doc_html/reference-function-ibeos.html
    Default value:1

CalculatedVariable

 A CalculatedVariable is a "synthetic" variable which is based on other "real" variables of the address-space. You can declare a CalculatedVariable under any object in the configuration file. The required attributes are:
  • name - will be the last part of the address
  • value - analytical formula used to calculate value of the variable. E.g. to calculate square of another variable with address x.y you should put x.y^2
Detailed documentation of all features of CalculatedVariables is available in quasar Documentation.
